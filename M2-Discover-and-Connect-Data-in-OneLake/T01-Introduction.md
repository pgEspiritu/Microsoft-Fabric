# 📘 Introduction to OneLake Data Discovery

## 🎯 Overview

Traditional analytics environments often store data across multiple storage accounts, databases, and file systems. Each team typically maintains separate copies of data, leading to:

- Data duplication
- Data inconsistency
- Increased storage costs
- Time wasted searching for data
- Difficult collaboration

Before building reports or transforming data, users must first discover what data exists and where it is located.

---

# 🌊 Microsoft Fabric and OneLake

With **Microsoft Fabric**, all analytics workloads are unified through **OneLake**, providing a centralized storage layer for organizational data.

Instead of searching across multiple systems, users can discover and access data from a single location.

### Benefits of OneLake

- Centralized data storage
- Unified access across Fabric workloads
- Reduced data duplication
- Easier collaboration
- Faster data discovery

---

# 🔍 Data Discovery Scenario

Imagine your organization has recently adopted Microsoft Fabric.

Different teams have already created:

- Lakehouses
- Data Warehouses
- Streaming Data Sources

Before building your own analytics solution, you need to:

1. Discover available data.
2. Locate where the data is stored.
3. Connect to existing datasets.
4. Reuse trusted organizational data.

# 📝 Key Takeaways

- Traditional analytics environments often suffer from scattered and duplicated data.
- OneLake provides a unified storage layer across Microsoft Fabric.
