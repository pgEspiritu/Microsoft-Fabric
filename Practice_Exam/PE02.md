# Question 11

### You have an active relationship between Date and OrderDateKey and an inactive relationship between Date and ShipDateKey. How should you create Sales Ordered and Sales Shipped measures?

✅ **Correct Answer**

```DAX
Sales Ordered =
SUM ( 'Sales'[SalesAmount] )


Sales Shipped =
CALCULATE (
    SUM ( 'Sales'[SalesAmount] ),
    USERELATIONSHIP ( 'Date'[DateKey], 'Sales'[ShipDateKey] )
)
```

### Explanation

The model has:

- Active relationship:
  - Date[DateKey] → Sales[OrderDateKey]
- Inactive relationship:
  - Date[DateKey] → Sales[ShipDateKey]

The active relationship works automatically for **Sales Ordered**.

The inactive relationship must be activated using:

```DAX
USERELATIONSHIP()
```

Why not `RELATED()`?

- `RELATED()` retrieves values from related tables.
- It does not activate relationships.

# Question 12

### You need to confirm that queries are mapped to aggregated data. Which two tools should you use?

Select two.

✅ **Correct Answers**

- **DAX Studio**
- **Performance Analyzer**

### Explanation

To validate whether Power BI is using aggregations:

### DAX Studio

Provides:

- Query plans
- Storage engine queries
- Aggregation table usage

### Performance Analyzer

Provides:

- Visual query performance
- DAX query details

Why not others?

| Tool | Purpose |
|------|---------|
| SQL Profiler | Mainly SQL Server tracing |
| Tabular Editor | Model editing |
| VertiPaq Analyzer | Model size/compression analysis |

# Question 13

### Customer status changes must be preserved and include timestamps for point-in-time analysis.

✅ **Correct Answer**

**Type 2 Slowly Changing Dimension (SCD)**

### Explanation

Type 2 SCD maintains historical records.

Example:

| Customer | Status | Effective Date |
|----------|--------|----------------|
| John | Active | Jan 2026 |
| John | Inactive | Mar 2026 |

Benefits:

- Historical reporting
- Point-in-time analysis
- Audit tracking

# Question 15

### Which DAX filter provides the best performance?

✅ **Correct Answer**

```DAX
CALCULATE(
    [Sales],
    Calendar[Year] = 2023
)
```

### Explanation

Filtering a **dimension table** is more efficient because:

- The filter propagates through relationships.
- The Storage Engine can optimize the query.
- The large Sales fact table is not scanned unnecessarily.

Avoid:

```DAX
YEAR(Sales[Date])
```

because applying functions to columns reduces optimization opportunities.

# Question 16

### You need to migrate a 200-million-row Snowflake table into Fabric Lakehouse using a pipeline. What is the fastest method?

Options:

- Data Pipeline (Copy data)
- Data Pipeline (Dataflow Gen2)
- Data Pipeline (Lookup)
- Data Pipeline Spark (Notebook)

✅ **Correct Answer**

**Data Pipeline (Copy data)**

### Explanation

The Copy Data activity is optimized for large-scale ingestion.

Advantages:

- Parallel data movement
- High throughput
- Direct source-to-target transfer

Why not Dataflow Gen2?

Dataflows are better for transformation workloads, not massive bulk migration.

# Question 17

### Import semantic model refresh takes more than three hours. Only new and updated rows should refresh.

✅ **Correct Answer**

**In Power Query, configure incremental refresh for the table.**

### Explanation

Incremental refresh:

- Creates partitions automatically.
- Refreshes only recent data.
- Keeps historical data unchanged.

Example:

Archive:

```
3 years
```

Refresh:

```
Last 7 days
```

# Question 18

### Delta table contains 1,000 small Parquet files. Improve performance and reduce storage while keeping 7-day history.

✅ **Correct Answer**

**Run OPTIMIZE and VACUUM with a retention policy of seven days.**

### Explanation

Two maintenance operations are needed:

## OPTIMIZE

Purpose:

- Combines small files.
- Improves query performance.

Before:

```
1000 files x 1 MB
```

After:

```
Fewer optimized files
```

## VACUUM

Purpose:

- Removes obsolete files.
- Reduces storage costs.

The seven-day retention protects required history.

# Question 19

### Dataflow Gen2 tables must be optimized for Direct Lake semantic models.

✅ **Correct Answer**

**Use OPTIMIZE to apply V-Order.**

### Explanation

V-Order improves Direct Lake performance by:

- Optimizing Parquet files.
- Improving compression.
- Improving read performance.

It is especially beneficial for:

- Fabric Lakehouse
- Direct Lake semantic models
