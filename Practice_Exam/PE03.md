# Question 21

### A Fabric warehouse query performs poorly because table statistics are outdated. Which column statistics should you update?

Options:

- VARCHAR and NVARCHAR columns
- GROUP BY columns
- ✅ **Columns with high cardinality**
- Primary key columns

## ✅ Correct Answer

**Columns with high cardinality**

### Explanation

Statistics help the SQL engine estimate:

- Number of rows
- Data distribution
- Query execution plans

Columns with **high cardinality** have many unique values and provide more useful information for query optimization.

Examples:

- CustomerID
- TransactionID
- OrderID

# Question 22

### A Dataflow Gen2 query is slow because filtering happens after splitting a DateTime column. What should you recommend?

Options:

- ✅ **Apply the filter transformation to OrderDateTime first, then split the column.**
- Split by delimiter instead.
- Use Table.Buffer.
- Remove OrderDateTime.2.

## ✅ Correct Answer

**Filter first, then split the column.**

### Explanation

This improves performance by applying **query folding**.

Current process:

```
SQL Source
 ↓
Split DateTime
 ↓
Filter Year
```

The entire 20-year dataset is processed.

Improved process:

```
SQL Source
 ↓
Filter current year
 ↓
Split DateTime
```

Only required rows are processed.

# Question 25

### Combine sales data from Warehouse2 with HR data from Warehouse1 while minimizing effort and data movement.

Options:

- Copy HR data using Dataflow Gen2
- Copy HR data using Pipeline
- Copy HR data using Spark
- ✅ **Use cross-database querying between Warehouse1 and Warehouse2**

## ✅ Correct Answer

**Use cross-database querying between Warehouse1 and Warehouse2**

### Explanation

Fabric warehouses support cross-database queries.

Benefits:

- No data duplication
- No ETL pipeline required
- Minimal development effort

Example:

```sql
SELECT *
FROM Warehouse2.dbo.Sales S
JOIN Warehouse1.dbo.Employee H
ON S.EmployeeID = H.EmployeeID
```

# Question 30

### Source table is truncated daily and replaced with current payments. You need to retain current and historical records using Dataflow Gen2.

Options:

- ✅ **Configure refresh to append data for the query**
- Replace data
- Optimize V-Order
- Ensure query folding

## ✅ Correct Answer

**Configure the refresh to append data for the query.**

### Explanation

The source system only keeps current data.

Using **append mode**:

- Adds new records
- Preserves previous records
- Builds historical data over time

Example:

Day 1:

```
Payment A
Payment B
```

Day 2 source replaces data:

```
Payment C
```

Append result:

```
Payment A
Payment B
Payment C
```
